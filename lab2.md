# 1. Dictionary of Basic English Terms

CLI (Command Line Interface) – A text-based interface used to interact with the operating system, where the user types commands via a keyboard to perform various tasks.  

Terminal – An application that provides access to the command line within a graphical Linux environment.  

Virtual Terminal – A text-only Linux mode that operates without a graphical interface, typically accessed via specific keyboard shortcuts (e.g., Ctrl+Alt+F1).  

Kernel – The core of the operating system that manages hardware resources and facilitates communication between software and hardware.  

Process – An instance of a running program or task being executed by the operating system.  

Shell – A command-line interpreter that takes user input and passes it to the system kernel (e.g., Bash, Zsh).

# 4. Визначення основних понять

CLI-режим це режим роботи операційної системи, у якому керування здійснюється шляхом введення текстових команд без використання графічного інтерфейсу.

# Відповіді на контрольні запитання

Приклади серверних додатків Linux
Сервери баз даних: mysql (найпопулярніша для веб-сайтів), postgresql (потужна об’єктно-реляційна база), mariadb (відгалуження mysql з відкритим кодом), mongodb (для неструктурованих даних).
Сервери розсилки повідомлень: postfix (висока продуктивність), exim (гнучкість конфігурації), dovecot (для отримання пошти через imap/pop3).
Файлообмінники та сховища: vsftpd (безпечний ftp), samba (реалізація протоколу smb для сумісності з windows), nextcloud (власна хмарна платформа для файлів).
2. Порівняння оболонок (Shells)
Bourne shell (sh): розроблена стівеном борном; це фундамент для написання системних скриптів, але вона не має історії команд або автодоповнення.
C shell (csh): створена біллом джоєм; її синтаксис нагадує мову c, що зручно для програмістів, але вона менш ефективна для автоматизації.
Bourne Again shell (Bash): розширена версія sh; підтримує історію команд, псевдоніми (aliases) та вбудовані арифметичні операції.
Tcsh: версія c shell з покращеним редагуванням рядків та обробкою помилок.
Korn shell (Ksh): розроблена девідом корном; поєднує математичні можливості csh та синтаксис sh.
Zsh: сучасна оболонка, що пропонує просунуте автодоповнення (навіть для аргументів команд) та підтримку тисяч плагінів через oh-my-zsh.
3. Менеджери пакетів
Менеджер пакетів потрібен для: розв'язання залежностей (коли одна програма вимагає встановлення іншої), перевірки цифрових підписів софту та централізованого оновлення всієї системи.
RPM-based: dnf (використовується у вашій fedora), yum (старіша версія dnf).
Debian-based: apt (найвідоміший через ubuntu), dpkg (низькорівневий інструмент).
Інші: pacman (arch linux), snap та flatpak (універсальні контейнеризовані пакети).
4. Засоби безпеки в Linux
Дискреційне керування доступом (DAC): стандартні права rwx (read, write, execute) для власника, групи та інших.
Мандатне керування доступом (MAC): selinux (security-enhanced linux) у вашій fedora створює жорсткі політики для кожного процесу.
Брандмауери: firewalld (графічний інтерфейс для керування мережевими правилами) та nftables.
SSH-ключі: використання криптографічних ключів замість паролів для віддаленого входу.
5. Актуальність віртуалізації
Віртуалізація стала критично важливою через:
Ізоляцію: помилка в одній віртуальній машині не впливає на основну систему або інші віртуальні машини.
Тестування: можливість безпечно запускати різні ос (як ви запустили linux на windows) для навчання чи розробки.
Знімки (Snapshots): можливість миттєво повернути систему до попереднього стану, якщо щось зламалося.
6. Поняття контейнеризації
Контейнеризація — це віртуалізація на рівні операційної системи. На відміну від віртуальних машин (VirtualBox), контейнери (Docker) не мають власного ядра; вони використовують ядро хоста, але мають власні ізольовані бібліотеки та файлову систему. Це робить їх у 10-100 разів легшими за віртуальні машини.
7. Переваги та недоліки ПЗ з відкритим кодом
Переваги: відсутність ліцензійних платежів, прозорість (немає прихованих шпигунських функцій), швидке виправлення багів спільнотою.
Недоліки: іноді "сирий" інтерфейс, відсутність персональної технічної підтримки (доводиться шукати відповіді на форумах), складність налаштування деякого обладнання через відсутність пропрієтарних драйверів.
8. Віртуальні консолі (TTY)
У Linux за замовчуванням працює 6 текстових терміналів.
Виклик: Ctrl + Alt + F1 — зазвичай графічне вікно або логін; Ctrl + Alt + F2-F6 — текстові консолі.
Приклад перемикання: якщо графічний інтерфейс завис, ви можете натиснути Ctrl + Alt + F3, увійти в систему як root і вбити процес, що спричинив збій, командою kill.
9. Графічна консоль
У сучасних дистрибутивах (як ваша Fedora 43) графічна оболонка зазвичай завантажується на TTY1 або TTY2. Раніше стандартом була сьома консоль (TTY7).
10. Реєстрація під одним ім'ям
Linux — це багатокористувацька та багатозадачна система. Ви можете увійти в систему 10 разів одночасно з одного акаунта.
Переваги: одночасне виконання фонових завдань, можливість моніторити систему через один термінал під час написання коду в іншому, віддалена допомога (адмін може зайти під вашим логіном по мережі, поки ви працюєте локально).
